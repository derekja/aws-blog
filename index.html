AWS Innovation Sandbox Distributed Systems Cloud Education

How Innovation Sandbox Facilitates Cloud Environment Learning in Distributed Systems Education

By Reed Huang, Derek Jacoby, and Yvonne Coady
September 14, 2025

Teaching distributed systems in university courses requires robust cloud environments that can accommodate complex student projects while maintaining cost control and administrative efficiency. This post explores how AWS Innovation Sandbox offers advantages over traditional AWS Learner Lab environments for academic distributed systems coursework.

Context & Challenges with AWS Learner Lab

To learn fundamental knowledge of distributed systems, courses guide students to deploy, interact, and test their services in AWS environments. AWS Academy's Learner Lab platform simplifies the workflow by providing federated accounts with fixed budgets and custom duration, but comes with significant limitations:

  • Budget Cap Restrictions: The $50 budget limit often proves insufficient for comprehensive distributed systems projects
  • Limited Service Access: Restricted range of AWS services available, limiting the scope of learning materials
  • IAM Limitations: Strict prohibition of user and role creation through the existing "LabRole", preventing realistic cloud architecture learning
  • Account Termination: Accounts are directly terminated when credits expire, forcing students to rebuild services
  • Limited Visibility: Restricted access to billing and cost pages makes it difficult for students to track usage

The restricted IAM access becomes problematic when incorporating advanced projects that require services beyond the approved list. In a recent distributed systems course with 30 students, more than a third required new account requests during the semester, creating significant administrative burden for both instructors and students.

AWS Innovation Sandbox: A Comprehensive Solution

AWS Innovation Sandbox, designed specifically for use cases like higher education training labs, addresses many of these limitations by providing a more flexible and comprehensive educational environment.

Enhanced Account Management

Innovation Sandbox offers sophisticated account lifecycle management through AWS Organizations, reducing administrative overhead. Accounts can be configured with both time-limited (in hours) and cost-limited (in integer dollars) constraints, with global values set via AppConfig or local values through lease templates in the web UI.

Broader Service Access and Permissions

Unlike Learner Lab's restrictive service list, Innovation Sandbox provides access to a comprehensive range of AWS services. Students can work with advanced networking, container orchestration (ECS/ECR), and serverless computing services essential for modern distributed systems education.

However, the default user permissions may require adjustment for certain services. For advanced projects requiring ECS/ECR deployment, Service Control Policies (SCPs) in AWS Organizations may need modification to grant appropriate access.

Improved Cost Monitoring and Budgeting

The platform provides enhanced visibility into account usage through integrated budget tracking and cost monitoring tools. While the system includes an alert system for notifications, budget UI updates and Cost Explorer data may have some refresh delays.

Administrative Control and Flexibility

The admin account provides comprehensive control over account lifecycle management, including moving accounts between states (entry to cleanup, active to frozen), lease template management, and user access control through Identity Center integration. The platform supports three user levels: admin, manager, and user, each with appropriate permissions for their role.

Educational Impact and Use Cases

The enhanced capabilities of Innovation Sandbox enable more sophisticated distributed systems projects, including:

  • Multi-region deployments for studying latency and consistency
  • Auto-scaling scenarios for load testing and performance analysis
  • Complex microservices architectures with proper security boundaries
  • Container orchestration and observability systems
  • Hybrid cloud and edge computing scenarios

Projects like observability systems in distributed environments become feasible with the expanded service access and flexible permission model that Innovation Sandbox provides.

Implementation Considerations

While Innovation Sandbox offers significant advantages, educators should consider several factors:

  • Setup Complexity: Initial configuration requires more planning than Learner Lab, with manual SAML 2.0 application creation steps that don't support automation
  • Permission Configuration: Default user permissions may require customization for specific course requirements
  • Monitoring Requirements: Enhanced capabilities require more sophisticated usage monitoring and cost management
  • Student Onboarding: Students may need additional training on the expanded service portfolio

Technical Setup Process

The deployment follows a structured four-step process outlined in the official AWS documentation:

  1. Deploy the AccountPool stack for account orchestration
  2. Deploy the IDC (Identity Center) stack for authentication management
  3. Deploy the Data stack for configuration and monitoring
  4. Deploy the Compute stack for the web interface and processing

Note that production-level access for SES (Simple Email Service) can take up to 2 days if experimenting with new domains for notification systems.

Looking Forward

As distributed systems education evolves to meet industry demands, platforms like AWS Innovation Sandbox provide the flexibility and comprehensive service access needed to prepare students for real-world challenges. The ability to work with production-grade tools and realistic constraints better prepares graduates for careers in cloud-native distributed systems development.

For educators currently using AWS Learner Lab, Innovation Sandbox presents a compelling upgrade path that addresses many common limitations while providing the administrative control necessary for effective course management. The enhanced capabilities and reduced long-term administrative overhead make it an attractive solution for advanced distributed systems education.

AWS Development Teams Claude Code Infrastructure as Code

Running development teams at UVic and with the Pacific Coastal Computing Association

By Derek Jacoby
September 8, 2025

Summertime is internship season at UVic! Our students go off to various companies, and they also work on joint research projects. One of the projects that we are working on is with a non-profit called the Pacific Coastal Computing Association who is developing an application called KnowledgeGraph to use AI to assist in educational efforts, both at UVic and other institutions, and in consultation with community efforts.

You can hear our development team describe one current project!

This work is also highly leveraging AWS infrastructure. Nathan Inbar is a UVic student that spent most of his summer wrangling various AWS services - EC2, S3, Application Gateway, Bedrock, ECR/ECS, etc - to create a cloud native version of KnowledgeGraph.

The team is using Claude Code as our AI tool of choice, and it helps to have some structure around the use of the tool to keep consistency. Even the markdown files give us a good idea of self-documenting features of using the AI tools, but the real benefit comes from allowing them to tune our working environment. We use the AI tools to plan development features, to start the development process, to do code reviews and test case development. One thing we do NOT do is vibe code. Every developer on the team is responsible for fully understanding and approving every line of code that goes into the product. How much of the code is automatically generated varies both by team member and application area.

Although we haven't made this shift yet, we are evaluating a tool that has just come out from Github called spec-kit. As we move towards a new phase of the project in the fall we are investigating spec-kit as an AI coding model to move towards.

The other work presently going on is to tie the AWS services into a Terraform based setup. This is one of the areas that is always a barrier to new people on the AWS platform - understanding the wealth of services and using the web console to set them up is just a very complex learning process. The use of Terraform allows us to use an Infrastructure as Code (IaC) approach to deployment that is really important in creating repeatable installations on AWS.

In one exciting development, one of our team members has been asked to speak at a major Git conference at the end of the month! Congratulations Piyush in this fun opportunity!

AWS SIGGRAPH AI Applications Claude Code

UVic at the ACM SIGGRAPH Appy Hour

By Derek Jacoby
August 14, 2025

Mid-August of 2025, a group of UVic students participated in SIGGRAPH's Appy Hour. They were demonstrating several applications that we built using Amazon Web Services to underlie the AI applications that we chose to engage audiences with. These were examples of the type of application that can support museum installations.

In the first example, Annie McClure built a web page and a series of speech interactive applications. She was experimenting with AI authoring tools and her first task was to figure out how to get Claude Code to work with the Anthropic models on AWS Bedrock! Fortunately, there were some guides to lead her way (Claude Code on Amazon Bedrock Quick Setup Guide).

Her next task was to have Claude help her develop some web pages. A theme of the exhibit was diatoms - microscopic ocean creatures that serve an important role in ecosystem signalling. There were also two other example applications that she worked on.

These web pages were backed up by a voice interactive system that operates over the phone lines. Please call it yourself! Dr. Marino Waters (an AI) can be reached at 778-817-1483.

Annie worked out illustrative prompts for that system, which is based on the AWS Nova Sonic speech to speech model and allows the system to provide snappy responses that mimic human-to-human conversations. These were tied in to the telephone network through a Voice over IP service (voip.ms) and the AWS Connect service.

Another student, Nathan Inbar, did work to get a demonstration of the diatom application to run on the Sony SRD monitor which Sony loaned to us to try developing some applications on.

Nathan and Hongyu Wan handled the crowds at SIGGRAPH with aplomb, answering questions and demonstrating the applications. Hongyu was the primary author of the Unity-based game segment that was shown to engage visitors with the world of diatoms. Here is a video of that application!

AWS Cloud Computing University Research

Exploring AWS Innovation: A University's Journey into Cloud Computing Excellence

By Reed Huang, Derek Jacoby, and Yvonne Coady
August 12, 2025

Welcome to our AWS Explorations Blog, where we document our university's journey into the vast landscape of Amazon Web Services. This platform serves as a comprehensive resource for educators, researchers, and students who are navigating the complexities of cloud computing in academic environments.

Our Mission

As educators and researchers in the rapidly evolving field of technology, we recognize the transformative potential of cloud computing in academic settings. Amazon Web Services offers an unprecedented array of tools and services that can revolutionize how universities approach teaching, research, and innovation.

Through this blog, we aim to share our hands-on experiences, practical insights, and lessons learned as we explore various AWS services and their applications in higher education. Our goal is to create a valuable resource that helps fellow educators harness the power of the cloud while navigating the unique challenges and opportunities present in academic environments.

What You'll Find Here

Our exploration will cover a wide range of AWS services and use cases relevant to university settings:

  • Educational Technology Solutions: How AWS services can enhance online learning platforms, student information systems, and collaborative research environments.
  • Research Computing: Leveraging AWS's computational power for data analysis, machine learning projects, and high-performance computing tasks.
  • Cost-Effective Infrastructure: Strategies for optimizing AWS usage to fit within university budgets while maximizing educational impact.
  • Security and Compliance: Ensuring data protection and meeting regulatory requirements in educational cloud deployments.
  • Innovation Sandboxes: Creating safe, controlled environments for students and faculty to experiment with cutting-edge technologies.

Join Our Journey

Whether you're a fellow educator looking to integrate cloud technologies into your curriculum, a researcher seeking scalable computing solutions, or an administrator exploring cost-effective IT infrastructure, we invite you to join us on this exploration.

Each post will provide practical insights, real-world examples, and actionable guidance based on our direct experience with AWS services. We believe that by sharing our discoveries, challenges, and successes, we can collectively advance the use of cloud computing in higher education.

Stay tuned for upcoming posts where we'll dive deep into specific AWS services, share implementation case studies, and provide step-by-step guides for common university use cases. Together, we'll explore how AWS can empower educational institutions to innovate, scale, and deliver exceptional learning experiences in the digital age.
